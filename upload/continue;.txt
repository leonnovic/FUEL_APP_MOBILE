## 3. CORE MODULES SPECIFICATION (UI & LOGIC)

### MODULE 1: Dashboard & Business Intelligence
*   **Logic:** Aggregates data from local DB. Calculates 7-day growth, net profit, and fuel mix (PMS vs AGO).
*   **UI Layout:** 
    *   **Header:** Station Selector (Dropdown with "Combined View"), User Profile, Sync Status Indicator (Green=Online, Red=Offline).
    *   **KPI Cards:** Total Revenue, Net Profit, Fuel Sold (Litres), Cash at Hand.
    *   **Charts:** Sales Trend (Line), Fuel Distribution (Pie), Expense Breakdown (Bar).
    *   **Widgets:** Current Pump Prices, Tank Levels (Visual gauges), Weather Widget (OpenWeather API), Regulatory Alerts (KRA/EPRA).

### MODULE 2: Point of Sale (POS) & KRA eTIMS
*   **Logic:** Quick entry for fuel sales. Automatically calculates 16% VAT. Generates ETR-compliant receipt strings: `ETR-[station]-[date]-[sequence]`.
*   **UI Layout:**
    *   **Quick Sale:** Fuel Type Selector (Petrol/Diesel), Litres Input, Auto-calculated Total.
    *   **Cart:** Custom items, Customer Name, KRA PIN (mandatory for B2B).
    *   **Payment:** Cash, M-PESA (Triggers Daraja STK Push), Card, Bank.
    *   **Receipt:** Auto-generates a printable thermal receipt format with KRA QR code data.

### MODULE 3: Sales Tracking & Shift Management
*   **Logic:** Reconciles physical pump meters with system sales. 
*   **UI Layout:**
    *   **Shift Selector:** Day / Night / Full Day.
    *   **Tank Inventory:** Opening/Closing meter readings for PMS and AGO tanks. Auto-calculates litres sold.
    *   **Pump Readings Grid:** Dynamic rows for multiple pumps per fuel type.
    *   **Daily Expenses:** Quick add for shift-specific expenses (e.g., generator fuel).
    *   **Till Reconciliation:** Input total Till/M-PESA payments. System calculates expected cash in hand and highlights discrepancies.

### MODULE 4: Inventory & Fuel Offloading
*   **Logic:** `Current Stock = Previous Stock + Offloaded - Sold`. Triggers alerts if `current_stock < alert_threshold`.
*   **UI Layout:**
    *   **Inventory Table:** Fuel Type, Stock (L), Capacity, Fill %, Unit Cost, Selling Price, Status Badge (Normal/Low/Critical).
    *   **Offloading Tracker:** Form to log truck deliveries (Truck Reg, Driver, Supplier, Quantity, Rate). Updates inventory automatically.

### MODULE 5: Finance (Payroll, Expenses, M-PESA Analyzer)
*   **Payroll Logic (Kenya Specific):** 
    *   `SHA` = 2.75% of Gross.
    *   `NSSF` = Tier I (6% of first 7,000) + Tier II (6% of 7,001-36,000).
    *   `PAYE` = Progressive tax bands (10% to 32.5%).
    *   **UI:** Data table showing Gross, SHA, NSSF, Advance, Net Pay. Bulk actions for "Edit SHA for All", "Export NSSF List".
*   **M-PESA Inflow Analyzer (AI Feature):**
    *   **Logic:** User uploads M-PESA PDF. System uses regex/AI to extract lines matching `Merchant Payment from`. Filters out loans, charges, and reversals.
    *   **UI:** Drag-and-drop PDF zone. Progress bar. 
    *   **Output:** Clean Revenue Breakdown cards (Valid Inflows vs Excluded). Reconciliation table matching M-PESA receipts to recorded POS sales. Discrepancy rate calculation.

### MODULE 6: Operations (Maintenance, Quality, Delivery)
*   **Maintenance:** Kanban/List view for equipment tasks (Pump calibration, Generator service). Status: Scheduled, In Progress, Overdue.
*   **Fuel Quality (EPRA Standards):** 
    *   Form to record batch tests. 
    *   **Validation Rules:** PMS Density (720-775 kg/m³), Sulfur (max 50ppm), Flash Point (min 38°C). AGO Density (820-860 kg/m³), Flash Point (min 52°C). Auto-fails if out of bounds.
*   **Delivery Tracker:** B2B fuel delivery log. Captures Client, Litres, Price, Vehicle Reg, Payment Status.

### MODULE 7: CRM, Credit & Invoicing
*   **Credit Accounts:** Track B2B customers. `Balance Used` vs `Credit Limit`. Block POS credit sales if limit exceeded.
*   **Invoice Generator:** Add line items, auto-calculate totals. "Edit Bank Details" modal to inject company bank info. Export to PDF/Excel.
*   **Debt Reminder:** Template builder for WhatsApp/Email/SMS. Select overdue clients, click "Send Reminders".

### MODULE 8: Reports Center
*   **Report Types:** Overall P&L, VAT Return (KRA), KRA Daily Sales, Fuel Sales Report (Monthly).
*   **UI:** Date range picker, Currency selector. 
*   **Output:** Visual breakdown of Revenue vs Costs. Export buttons for PDF, Excel, and direct KRA eTIMS submission payload generation.

**Kenya Payroll Calculations:**
```typescript
// SHA (Social Health Authority) - 2.75% of gross
const calculateSHA = (grossPay: number) => grossPay * 0.0275;

// NSSF (National Social Security Fund) - Tier I: 6% of first 7,000, Tier II: 6% of 7,001-36,000
const calculateNSSF = (grossPay: number) => {
  const tier1 = Math.min(grossPay, 7000) * 0.06;
  const tier2 = Math.max(0, Math.min(grossPay - 7000, 29000)) * 0.06;
  return tier1 + tier2;
};

// PAYE (Pay As You Earn) - Progressive tax rates
const calculatePAYE = (taxableIncome: number) => {
  if (taxableIncome <= 24000) return taxableIncome * 0.10;
  if (taxableIncome <= 32333) return 2400 + (taxableIncome - 24000) * 0.25;
  if (taxableIncome <= 50000) return 4458 + (taxableIncome - 32333) * 0.30;
  return 9813 + (taxableIncome - 50000) * 0.325;
};

const calculateNetPay = (employee: Employee) => {
  const gross = employee.basic_salary + employee.allowances;
  const sha = calculateSHA(gross);
  const nssf = calculateNSSF(gross);
  const paye = calculatePAYE(gross - nssf);
  const deductions = sha + nssf + paye + employee.advance_balance;
  return gross - deductions;
};
```

**PDF Extraction Logic:**
```typescript
const extractMpesaInflows = async (pdfBuffer: Buffer) => {
  // Use pdf-parse library
  const pdfData = await pdfParse(pdfBuffer);
  const text = pdfData.text;
  
  // Pattern matching for "Merchant Payment from"
  const merchantPattern = /Merchant Payment from\s+([\d\*]+)-([A-Z\s]+)/g;
  const amountPattern = /Ksh\s+([\d,]+\.\d{2})/g;
  const datePattern = /(\d{4}-\d{2}-\d{2}\s+\d{2}:\d{2}:\d{2})/g;
  const receiptPattern = /([A-Z]{2}\d+[A-Z0-9]+)/g;
  
  const inflows = [];
  let match;
  
  while ((match = merchantPattern.exec(text)) !== null) {
    const phone = match[1];
    const name = match[2].trim();
    
    // Extract amount (next occurrence after this match)
    const amountMatch = amountPattern.exec(text.substring(match.index));
    const amount = amountMatch ? parseFloat(amountMatch[1].replace(',', '')) : 0;
    
    // Extract date
    const dateMatch = datePattern.exec(text.substring(match.index));
    const date = dateMatch ? dateMatch[1] : '';
    
    // Extract receipt number
    const receiptMatch = receiptPattern.exec(text.substring(match.index));
    const receipt = receiptMatch ? receiptMatch[1] : '';
    
    inflows.push({
      details: `Payment from ${phone}-${name}`,
      paid_in: amount,
      date,
      receipt_number: receipt,
      type: 'merchant_payment'
    });
  }
  
  // Filter out loans, charges, transfers
  const validInflows = inflows.filter(inflow => 
    !inflow.details.includes('Loan') &&
    !inflow.details.includes('Charge') &&
    !inflow.details.includes('Transfer')
  );
  
  return validInflows;
};
```
